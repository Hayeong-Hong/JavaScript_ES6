1. ES5 var의 문제점
     - var는 중복선언이 가능함
     - var는 블록단위의 변수가 아님(함수나 전역단위임)
     - var는 호이스팅이 됨(선언순서 끌어올림)

2. 블록레벨 단위의 let 등장
     - let은 중복선언이 불가능함
     - let은 블록단위의 변수
     - let은 호이스팅도 불가능

3. 상수를 정의하는 const 키워드
     - 상수를 정의할 때는 선언과 동시에 초기화를 해줘야 함
     - let과 동일하게 블록단위의 변수이며, 호이스팅을 지원하지 않음
     - const는 상수이기 때문에 새로운 값을 할당할 수 없음(java의 final개념)
     - const를 선언할 때 변수명은 대문자로 지정한다.

4. const 키워드과 객체
     - const로 객체를 선언했을 때 객체안에 내용은 변경 가능
     - 원시값인 변수는 재할당없이는 변수 값을 변경할 수 없음
     - 객체의 속성값들은 재할당없이도 직접 변경할 수 있음
     - const는 재할당을 금지할 뿐이지 불변을 의미하지는 않는다.

5. ES6에서의 변수 선언
     - 기본적으로 const로 변수 선언하되, 재할당이 필요한 변수는 let으로 선언함
     - var를 사용하지 않음
     - let 변수를 선언할 때는 최대한 단위를 작게해서 선언
     - 상수와 객체는 최대한 const로 선언하도록 함
